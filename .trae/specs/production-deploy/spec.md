# Notifuse 生产环境部署规范

## Why
需要将 Notifuse 项目按照生产环境配置部署，使用正式的服务器 IP 地址，而不是本地开发配置。

## What Changes
- 修改后端配置以适配生产环境 IP
- 修改前端 Vite 配置以适配生产环境
- 确保所有服务在正式服务器 IP 上运行
- 修复 LOGO 显示问题
- 修复页面一直加载中的问题

## Impact
- 受影响的服务：前端、后端、数据库
- 受影响的文件：.env、vite.config.ts、nginx 配置（如有）

## 环境配置要求

### 后端配置 (.env)
```
SERVER_HOST=0.0.0.0
SERVER_PORT=8080
API_BASE_URL=http://<服务器IP>:8080
```

### 前端配置
- Vite 开发服务器监听所有网络接口
- API 代理指向正确的后端地址

### 生产环境部署检查项
- [ ] 后端服务运行在 8080 端口
- [ ] 前端服务运行在 3000 端口
- [ ] 数据库连接正常
- [ ] 所有 API 端点可访问
- [ ] 页面加载时间 < 10 秒

## 修复问题清单
1. LOGO 图片损坏问题
2. 页面无限加载问题
3. API 代理连接问题
4. Vite 配置代理问题
